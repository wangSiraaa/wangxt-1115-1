# 连锁门店临期商品调拨系统

## 系统概述

本系统实现连锁门店临期商品的调拨管理流程，涉及店长、区域仓和财务三个角色的协同作业。

## 业务流程

1. **店长**：提交临期商品清单（系统自动排除已过期商品）
2. **区域仓**：查看临期清单，安排调拨去向，选择运输车辆（冷藏商品强制冷链车辆）
3. **财务**：核算折扣损耗金额，完成结算（结算完成后损耗金额不可修改）

## 角色与权限

| 角色 | 权限 |
|------|------|
| 店长 | 查看库存、创建/编辑临期清单 |
| 区域仓 | 查看临期清单、安排调拨、选择车辆 |
| 财务 | 查看调拨单、核算损耗、完成结算 |

## 业务规则

- 已过期商品不能加入临期清单，系统自动过滤
- 含冷藏商品的调拨单必须选择冷链车辆
- 结算完成后，损耗金额锁定不可修改
- 库存批次全程跟踪（生产日期、保质期、库存数量）

## 快速开始

### 本地开发

```bash
# 安装依赖
npm install

# 启动前后端
npm run dev
```

- 前端：http://localhost:3000
- 后端：http://localhost:3001

### Docker 部署

```bash
# 启动服务
npm run docker:up

# 停止服务
npm run docker:down
```

访问 http://localhost:3000 即可使用系统，系统已预置演示数据。
